# RAG Knowledge Bot — 需求文档

## 功能目标

构建基于 RAG（检索增强生成）的内部知识库问答 API 服务，支持本地多格式、多语言文档导入、向量化存储与智能问答。

## 功能边界

| 做什么 | 不做什么 |
|:------|:--------|
| 多格式文档导入解析（PDF / Word / Markdown / TXT / Excel / CSV） | 用户认证鉴权 |
| 多语言支持（中文 / 英文 / 日文） | 前端 Web 界面 |
| 文本分块与向量嵌入存储 | 文档在线编辑/预览 |
| 基于语义相似度的文档检索 | 多租户隔离 |
| LLM 生成问答（可插拔：Ollama / 云端 API） | 实时文档同步 |
| REST API 接口 | |

## 验收标准

1. 调用文档上传 API 上传 PDF / Word / Markdown / TXT / Excel / CSV 文件 → 文档被解析、分块、向量化并存入向量库，返回成功状态
2. 调用问答 API 发送问题 → 返回基于知识库内容的回答，包含引用来源信息
3. 修改配置切换 LLM 后端（Ollama ↔ 云端 API）→ 系统正常工作，问答功能不受影响
4. 文档规模达 500 份时 → 检索响应时间 < 3 秒
5. 调用知识库管理 API → 可查看已导入文档列表、删除指定文档
6. 上传中文 / 英文 / 日文文档后提问 → 均能正确检索并生成对应语言的回答

## 约束条件

- Python 技术栈，FastAPI 框架
- 基于 LangChain 框架构建 RAG 管线
- 按中型规模（100~1000 份文档）设计，架构预留大型扩展能力
- LLM 可插拔，默认 Ollama 本地模型，预留云端 API 接口
- 嵌入模型需支持中英日多语言语义（BAAI/bge-m3）
