## Week 30 (D204-D210)：多模态 — 语音（ASR/TTS）

### D204 周一：多模态概览（模态/对齐/在 Agent 的角色）（2h）

**学习内容**：
- 多模态定位：文本、语音、图像、视频等模态的统一表征与互相转换，让 Agent 从「纯文本对话」扩展到「听说看画」
- 模态对齐核心思想：不同模态映射到共享语义空间（如 CLIP 的图文对齐），实现跨模态检索与理解的基础
- 多模态在 Agent 中的三类角色：感知输入（语音转文字、图像理解）、生成输出（文字转语音、文生图）、跨模态推理（看图回答、图文混合任务）
- 多模态架构范式：编码器拼接（各模态独立编码后融合）vs 原生多模态大模型（统一 Transformer 处理多模态 token）
- 工程接入方式：多模态能力作为独立微服务/插件挂载到 Agent，通过工具调用或前置/后置处理介入对话链路
- 成本与延迟意识：多模态模型普遍参数大、显存占用高、推理慢，需提前规划 GPU 资源与异步化处理

**检查点**：
- [ ] 能用自己的话说清「模态对齐」要解决的核心问题
- [ ] 能列出多模态能力接入 Agent 的三种角色及对应场景
- [ ] 理解多模态服务对 GPU 显存与推理延迟的额外要求

### D205 周二：ASR 语音识别（Whisper/实时/流式）（2h）

**学习内容**：
- ASR 定位：自动语音识别（Automatic Speech Recognition），将音频波形转写为文本，是语音 Agent 的输入入口
- Whisper 模型家族：OpenAI 开源的多语言 ASR，tiny/base/small/medium/large 各档在精度与速度上的权衡
- 加速实现：faster-whisper（CTranslate2 后端）相比原版的显存占用与速度优势，适合本地 GPU 部署
- 离线 vs 流式识别：整段音频一次转写（高精度、高延迟）vs 边说边转写（低延迟、需分块与上下文拼接）
- 流式分块策略：固定窗口分块、重叠窗口防止边界丢字、增量结果合并与去重
- 关键质量因素：采样率（16kHz 标准）、降噪、语言指定（自动检测 vs 显式指定）、热词与领域术语适配

**检查点**：
- [ ] 能说清 Whisper 不同尺寸模型在精度/速度/显存上的取舍
- [ ] 理解离线识别与流式识别的延迟与精度差异
- [ ] 能给出流式分块时防止边界丢字的处理思路

### D206 周三：TTS 语音合成（Edge-TTS/CosyVoice/流式播放）（2h）

**学习内容**：
- TTS 定位：文本转语音（Text-To-Speech），将 Agent 文本回复合成为自然语音，是语音 Agent 的输出出口
- 云端轻量方案：Edge-TTS 调用微软在线语音，零部署、多音色、低延迟，适合快速原型与生产兜底
- 开源高质量方案：CosyVoice / GPT-SoVITS 等支持音色克隆与情感控制的本地模型，可控但需 GPU
- 合成质量维度：自然度、韵律、停顿、多音字处理、数字与英文混读、情感语气
- 流式播放：按句/按段边合成边播放，降低首音延迟（TTFB），配合前端音频流式拉取
- SSML 与控制：通过 SSML 标记控制语速、音调、停顿、强调，提升合成可控性

**检查点**：
- [ ] 能对比 Edge-TTS（云端）与 CosyVoice（本地）的取舍场景
- [ ] 理解流式合成播放如何降低首音延迟
- [ ] 知道用 SSML 或参数控制语速/停顿/情感的基本方式

### D207 周四：语音 Agent 链路（ASR→LLM→TTS/打断/VAD）（2h）

**学习内容**：
- 语音 Agent 完整链路：麦克风采集 → VAD 检测 → ASR 转写 → LLM 推理 → TTS 合成 → 扬声器播放的闭环
- VAD（语音活动检测）：识别「有人在说话」的时间段，自动切分语句边界，过滤静音与噪声，触发 ASR
- 端点检测（Endpointing）：判断用户「说完了」的时机，平衡过早截断与等待过久，影响交互流畅度
- 打断（Barge-in）机制：用户说话时实时停止 TTS 播放并重新进入聆听，模拟自然对话的可打断性
- 全链路延迟优化：各环节流式串联（ASR 流式 → LLM 流式 → TTS 流式），用首字/首音延迟衡量体验
- 链路异常处理：识别为空、超时、LLM 故障、TTS 失败的降级（重听、提示、文本兜底）

**检查点**：
- [ ] 能画出语音 Agent 的端到端链路与各环节职责
- [ ] 理解 VAD 与端点检测在交互流畅度中的作用
- [ ] 能设计「用户打断正在播放的回复」的处理逻辑

## Week 31 (D211-D217)：多模态 — 图像生成

### D211 周一：图像生成原理（扩散模型/SD 基础）（2h）

**学习内容**：
- 扩散模型核心思想：前向过程逐步加噪、反向过程逐步去噪，从随机噪声生成图像的概率建模
- Stable Diffusion 架构：在隐空间（Latent Space）做扩散——VAE 编解码 + U-Net 去噪 + 文本编码器（CLIP）条件引导
- 采样器（Sampler）：DDIM / Euler / DPM++ 等采样算法在质量与步数（速度）上的权衡
- 关键参数：采样步数（steps）、提示词引导强度（CFG Scale）、种子（seed）对生成结果的影响
- 提示词工程：正向/负向提示词、权重语法、风格关键词，对生成质量与方向的控制
- 模型版本生态：SD 1.5 / SDXL / SD3 / FLUX 在质量、分辨率、显存需求上的差异

**检查点**：
- [ ] 能用自己的话解释扩散模型的加噪/去噪过程
- [ ] 说清 SD 在隐空间做扩散相比像素空间的优势
- [ ] 理解 steps / CFG / seed 三个参数对生成的影响

---

### D212 周二：SD 实战（文生图/图生图/ControlNet/LoRA）（2h）

**学习内容**：
- 文生图（txt2img）：纯文本提示词生成图像，提示词构造与参数调优的基本流程
- 图生图（img2img）：以参考图为起点重绘，重绘幅度（denoising strength）控制保留与改动的平衡
- ControlNet：用边缘/姿态/深度/线稿等条件图精确控制生成结构，实现可控构图与姿态
- LoRA：低秩适配的轻量微调，用小数据注入特定风格/角色/概念，加载即用、可叠加
- Inpainting / Outpainting：局部重绘与画面扩展，用于修补、替换、扩图等编辑场景
- 模型与权重管理：Checkpoint、LoRA、VAE、ControlNet 模型的组织与显存占用考量

**检查点**：
- [ ] 能说清文生图与图生图的差异及重绘幅度的作用
- [ ] 理解 ControlNet 如何用条件图控制生成结构
- [ ] 理解 LoRA 作为轻量微调注入风格/概念的原理

### D214 周四：视觉理解（多模态 LLM/图像问答/OCR）（2h）

**学习内容**：
- 视觉理解定位：让模型「看懂」图像内容——描述、问答、定位、推理，与图像生成方向相反

- OCR 能力：文档/截图/票据中的文字提取，VLM 端到端 OCR vs 专用 OCR 引擎（PaddleOCR）的取舍
- 文档理解：表格、版面、图表的结构化理解，从图像中提取结构化信息
- 接入方式：图像编码与上传、多模态 prompt 组装、与既有文本 Agent 的融合

**检查点**：
- [ ] 能说清多模态 LLM 如何把图像融入语言推理

### D220 周三：多模态评估与成本（质量/延迟/成本权衡）（2h）

**学习内容**：
- 评估维度：语音（识别准确率/合成自然度/首音延迟）、识图（VQA 准确率/OCR 字准）、生图（美学/一致性/提示遵从）
- 评估方法：人工评分 + 自动指标（WER、CLIP Score、相似度），以及小规模标注集与回归对照
- 延迟剖析：各模态各环节耗时拆解（ASR/VLM/SD 推理、排队、传输），定位体验瓶颈
- 成本剖析：GPU 占用与时长、生图单张成本、VLM/ASR 调用成本，自部署 vs API 的取舍
- 质量-延迟-成本三角：在体验目标下选择模型档位、采样步数、缓存策略的权衡决策
- 评估闭环：固化评估集与基线，每次改动复评，量化质量/延迟/成本的变动

**检查点**：
- [ ] 能为语音/识图/生图分别列出关键评估指标
- [ ] 能拆解多模态链路的延迟与成本构成
- [ ] 能在质量-延迟-成本三角上给出权衡决策
