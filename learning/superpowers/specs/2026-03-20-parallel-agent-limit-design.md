# 设计文档：dispatching-parallel-agents 最大并行数控制

**日期：** 2026-03-20
**状态：** 已批准
**目标：** 通过 CLAUDE.md 全局约束，限制 `dispatching-parallel-agents` 技能的最大并行 agent 数量，降低 API 调用开销，节省成本。

---

## 背景

`dispatching-parallel-agents` 技能在面对多个独立任务时会无上限地并行创建子 agent，在任务数量较多时会产生大量 API 调用开销。

---

## 方案

在全局 `~/.claude/CLAUDE.md` 的 `## 约束` 章节追加规则，由 Claude 在执行前自行遵守。

**无需修改技能文件**，改动最小，立即生效。

**约束性质说明：** 此为 prompt 级软约束，依赖 Claude 自觉遵守，无运行时强制机制。调整上限只需修改规则中的数字 `3`，无需其他改动。

**生效范围：** 全局生效，影响所有项目中 `dispatching-parallel-agents` 技能的调用。

---

## 配置规则

在 `~/.claude/CLAUDE.md` 的 `## 约束` 末尾追加以下规则：

```markdown
- 使用 `dispatching-parallel-agents` 时，根据任务复杂度自行判断并行数量，但每批并行 agent 数不得超过 **3**。若任务总数超过上限，按批次顺序执行，每批不超过该上限。如需调整上限，直接修改此处数字即可。
```

---

## 动态判断规则

Claude 在 dispatch 前评估任务复杂度，**按优先级从高到低**依次匹配，取第一个符合条件的规则：

| 优先级 | 任务特征 | 建议并行数 |
|:------:|---------|:---------:|
| 1（最高）| 任务复杂、需要大量上下文理解，或任务间存在隐式依赖 | 1（顺序执行）|
| 2 | 任务涉及多文件或有轻度关联 | 2 |
| 3（默认）| 每个任务仅涉及 1-2 个文件，逻辑完全独立 | 3 |

---

## 超限批次策略

当独立任务总数 > 3 时：

1. 将任务按独立性分组，每批 ≤ 3 个
2. 并行执行第一批，等待全部完成
3. 检查变更冲突：查看各 agent 是否修改了相同文件；若有冲突，手动合并后再继续
4. 若某批次中有 agent 失败：记录失败任务，完成当前批次其余任务后，将失败任务作为独立批次重新执行
5. 执行下一批，循环直到全部完成
