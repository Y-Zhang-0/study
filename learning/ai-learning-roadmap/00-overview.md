## 里程碑详情

### 里程碑 1（Week 1 末）
**产出**：能用 Python 写一个读取文件、调用网络接口的命令行程序
**验收标准**：
- [ ] 能安装并管理 Python 虚拟环境
- [ ] 能用 `requests` 库调用任意公开 API
- [ ] 能处理 JSON 数据并格式化输出

### 里程碑 2（Week 3 末）
**产出**：一个支持多轮对话的命令行聊天机器人
**验收标准**：
- [ ] 调通大模型 API（OpenAI / 国内模型）
- [ ] 支持流式输出（逐字显示）
- [ ] 理解并能调整 System Prompt
- [ ] 能设计 Few-shot Prompt 让模型按格式回复

### 里程碑 3（Week 7 末）
**产出**：RAG API + Redis 缓存就绪
**验收标准**：
- [ ] 支持上传 PDF/Word/MD 文档
- [ ] 能基于文档内容准确回答问题
- [ ] 有 FastAPI 接口，能用 Postman 调用
- [ ] Redis 缓存系统集成完成

### 里程碑 4（Week 9 末）
**产出**：RAG API 生产就绪（容器化、测试、部署）
**验收标准**：
- [ ] Docker 镜像可以运行
- [ ] 有基本的认证（API Key）
- [ ] 部署到云服务器可访问
- [ ] 知道如何分析和改善检索效果

### 里程碑 5（Week 12 末）
**产出**：Research Agent + Celery 异步化
**验收标准**：
- [ ] Agent 能自主决定调用哪些工具
- [ ] 能处理多步骤任务（搜索→分析→整理→输出）
- [ ] 有异常处理，不会无限循环
- [ ] 文档摄入异步化完成

### 里程碑 6（Week 14 末）
**产出**：并发优化完成
**验收标准**：
- [ ] 理解 asyncio/threading/multiprocessing 的适用场景
- [ ] 批量文档处理性能提升明显
- [ ] 有性能压测数据对比
- [ ] 掌握连接池和并发限制

### 里程碑 7（Week 16 末）
**产出**：掌握分布式架构设计
**验收标准**：
- [ ] 能解释 CAP 定理和无状态服务设计
- [ ] 理解分布式锁和负载均衡
- [ ] 能画出高并发 AI 服务架构图
- [ ] 完成后端架构设计题训练

### 里程碑 8（Week 20 末）
**产出**：1-2 个完整作品集项目，可对外展示
**验收标准**：
- [ ] 有完整的 README
- [ ] 代码托管在 GitHub
- [ ] 能向别人讲清楚技术实现
- [ ] 能在白板上画出高并发 AI 服务架构图

## 全栈开发者优势盘点

### 你已经有的能力

| 你的旧技能 | 对应的 AI 开发能力 | 转化成本 |
|:---|:---|:---|
| Node.js REST API 开发 | FastAPI 服务开发（换语法，逻辑相同） | 极低 |
| Spring Boot 分层架构 / 依赖注入 | AI 服务的分层设计（Config / Service / API） | 极低 |
| 前后端全栈经验 | 独立交付完整 AI 产品 | 无需转化 |
| HTTP 请求 / JSON 处理 | 调用 LLM API（本质就是一个 HTTP POST） | 极低 |
| 工业领域知识（AGV）| 垂直行业 AI 落地的场景感，作品集差异化 | 无需转化 |

### AI 应用开发的技能优先级

| 技能 | 优先级 | 说明 |
|:---|:---|:---|
| LLM API 调用 + Prompt 工程 | ★★★★★ | 核心中的核心 |
| RAG 系统（向量检索、分块、重排序）| ★★★★★ | 企业需求最大的场景 |
| Agent 框架（LangGraph / 自建）| ★★★★☆ | 进阶必备 |
| Python 工程化（FastAPI / async）| ★★★★☆ | 有 Node.js 底子，学习曲线平缓 |
| Docker 容器化部署 | ★★★☆☆ | 交付上线必须 |
| 多线程 / 高并发 / 分布式 / 消息队列 | ★★☆☆☆ | **规模化阶段才需要**，不是入门门槛 |
| 缓存 / 数据库调优 | ★★☆☆☆ | 同上，后期按需补充 |

> **关键认知**：AI 应用开发工程师 ≠ 后端架构师。没有分布式/高并发经验不是短板，只是参照系用错了。
> 当应用规模增长时，再按需学习分布式知识即可。
